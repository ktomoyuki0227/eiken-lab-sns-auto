# Tavily Search 活用リサーチ資料

作成日：2026-04-06
対象プロジェクト：英検ラボ SNS 自動投稿システム
ステータス：採用候補（確定ではない）

---

## 1. Tavily Search とは

Tavily は「AI エージェント向け」に特化した Web 検索 API。
通常の検索エンジンと違い、検索結果をそのままページリンクで返すのではなく、各ページの本文内容を要約・抽出して返してくれる。LLM との組み合わせに最適化されている。

主な特徴：
- 検索クエリに対して、各ページの要約テキストを返す（スクレイピング不要）
- リアルタイムの Web 情報を取得できる
- Dify に標準搭載されているため、ノーコードで使える
- 無料枠：月1000クエリ

---

## 2. Dify での使い方

Dify のワークフローに「Tools → Tavily」ノードを追加するだけで使える。
設定項目は以下のみ：

| 設定項目 | 内容 |
|---------|------|
| API キー | Tavily サイトでアカウント作成→無料で発行 |
| 検索クエリ | 前のノードから変数で渡すか、固定テキストで指定 |
| 検索件数 | 1〜10件（多いほどトークン消費が増える） |
| 検索深度 | basic（速い・軽い） / advanced（精度高い・遅い） |

返ってくるデータの形式（例）：
```json
{
  "results": [
    {
      "title": "英検2026年度第1回試験の申し込み受付開始",
      "url": "https://...",
      "content": "英検公式サイトによると、2026年度第1回試験の..."
    },
    ...
  ]
}
```

この `content` 部分が後続の LLM ノードに渡される。

## 3. 本プロジェクトでの活用案

### 検索クエリの設計

毎日の生成ワークフローで以下のクエリを実行する。

| 目的 | 検索クエリ例 |
|------|------------|
| 最新ニュース | 「英検 ニュース {今日の日付}」 |
| 試験情報 | 「英検 試験日 申し込み 2026」 |
| 勉強法トレンド | 「英検 勉強法 おすすめ」 |
| 社会トレンドと英語 | 「英語 学習 トレンド 2026」 |

Dify の変数機能で今日の日付を自動挿入できる。
複数クエリを並列実行し、結果をまとめて LLM に渡す構成が理想的。

### ワークフロー上の位置づけ

```
Dify スケジュールトリガー
    ↓
Tavily Search（クエリ1：英検ニュース）
Tavily Search（クエリ2：勉強法トレンド）  ← 並列実行
    ↓
結果をまとめて LLM ノードへ渡す
    ↓
Claude Haiku でコンテンツ生成
```

## 4. 無料枠の試算

| 項目 | 数値 |
|------|------|
| 1日あたりのクエリ数 | 2〜3クエリ |
| 月あたりの消費 | 60〜90クエリ |
| 無料枠（月1000クエリ） | 余裕あり（10倍以上の余裕） |

無料枠で十分に収まる。超過時は月$5〜$20程度のプランで対応可能。

---

## 5. メリット・デメリット

| メリット | デメリット |
|---------|-----------|
| Dify 標準搭載でノーコード設定が可能 | Tavily の API キー取得が別途必要（無料） |
| スクレイピング不要・サイト構造の変化に強い | 検索結果の質はクエリ設計に依存する |
| リアルタイムの情報が取れる | advanced モードは応答が少し遅い（3〜5秒） |
| 返却形式が LLM 向けに最適化されている | 無料枠を超えると有料（月数百円〜） |
| 英検公式X など X の投稿も検索対象に含まれる | 日本語コンテンツの品質は英語より若干落ちる場合あり |

---

## 6. 代替案との比較

| 方法 | 特徴 | 今回の適合度 |
|------|------|------------|
| Tavily Search | AI 向け検索 API。ノーコードで Dify 連携可能 | ◎ |
| Dify HTTP ノード + Jina Reader | 特定 URL の本文を抽出。対象 URL が固定の場合に有効 | ○（参照先URL確定後に追加で使える） |
| RSS フィード取得 | 英検公式などが RSS を提供していれば使える | △（英検公式の RSS は不明） |
| Google Custom Search API | Google の検索結果を取得。精度高い | △（設定が複雑、無料枠が少ない） |

---

## 7. 推奨実装方針

1. Tavily Search をメインの情報収集手段として採用
2. 検索クエリは 2〜3本を並列実行（英検ニュース・勉強法・試験日程）
3. 長畑さんから参照サイト URL が届いたら、その URL を Jina Reader で追加取得する構成を加える
4. 検索深度は basic から始め、品質が不十分なら advanced に切り替える
